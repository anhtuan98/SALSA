The blending of IoT and cloud technologies are emerging as a top research and development trend. IoT Cloud Systems are constructed by different types of devices and services, which are deployed on multiple IoT and Cloud infrastructure. To managing these systems, we need to perform various types of configuration such as provisioning infrastructures, deploying artifacts, orchestrating the configurations, wiring application components, and reconfiguring the system at runtime. To reduce the efforts of complex management and provide fine-grained configuration features, we develop SALSA, a framework for dynamically configuring IoT Cloud Systems on IoT and multi-cloud infrastructures.

Features:
  • Configure by multiple stacks: e.g. infrastructure, container/middleware, application, IoT unit.
  • Configure by multiple abstraction levels: e.g instance, instance-type, topology, whole system.
  • Configure by multiple phases: e.g. deployment time and runtime.
  • Configure on multiple clouds: run with both private and public cloud, or a multiple layers of docker platform for simulating cloud environments.
  • Configure with rich capabilities: SALSA's default capabilities and external capabilities.
  • Ensure configuration dependencies: wire the configurations, maintain the component lifecycle.
Usability and performance:
  • Perform the configuration by distributed and independent agents.
  • Orchestrate the agents and configuration processes centrally.
  • Collect more capabilities dynamically when external stakeholders involve.
  • Communicate via message queue: loose-couple management and error-tolerance.
Development:
  • Opensource
  • Pluggable design for extending the configuration mechanisms (e.g. orchestration algorithms, configuration qualities)
  • Pluggable design for extending the supported configuration (e.g. cloud systems, artifact types)
