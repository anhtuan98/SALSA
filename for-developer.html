Create new collector for data retrieving

An adaptor is an .jar package that is plugged in to the collector to query data. Followings are some guidelines and notations to create this:
  1. Create a normal Java project, then a class which extends the UnitInstanceCollector interface. The class should implement the mechanism (1) to collect the information and (2) to identify the instance.
  2. The collector may gather application-specific information, which should be marshalled into string-based format (e.g. can be JSON or XML) in order to store in the UnitInstance object.
  3. The implementation can reuse the function "readAdaptorConfig(String parameter)" to retrieve the value in properties file going with the plug-in.
  4. SALSA will merge the collected information with existed instances in the database, thus the identification must enable SALSA to link with existed information.
The artifact of the adaptor will be plugged in the collector as the previous guide. Please check the adaptor written for GovOps service. Other available adaptors are here.
